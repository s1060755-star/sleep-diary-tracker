# 🌙 Sleep Diary Tracker
## Щоденник сну — консольна утиліта для обліку тривалості сну
## 📌 Опис проєкту
**Sleep Diary Tracker** — це консольна програма, розроблена мовою Python, яка дозволяє користувачці вести щоденник сну.
Програма дає змогу:
- фіксувати час засинання та пробудження;
- автоматично розраховувати тривалість сну;
- переглядати всі записи у вигляді списку;
- аналізувати середню тривалість сну за останні 7 днів;
- визначати найкоротшу ніч за тиждень;
- редагувати та видаляти записи.
Усі дані зберігаються у файлі `sleep_data.json`, тому інформація не втрачається після закриття програми. Програма коректно обробляє помилки введення та не завершується аварійно при некоректних даних.

## 🎯 Функціонал програми
| №   | Функція                           |                                           Опис                                                                        |
|-----|---------------------------------- |-----------------------------------------------------------------------------------------------------------------------|
| 1️⃣ | **Додати запис сну**              | Користувачка вказує дату, час засинання та час пробудження. Програма автоматично розраховує тривалість сну в годинах.  |
| 2️⃣ | **Переглянути всі записи**        | Виводиться список усіх збережених записів, відсортованих за датою. Показується загальна та середня тривалість сну.     |
| 3️⃣ | **Середня тривалість за тиждень** | Аналізуються записи за останні 7 днів. Виводиться кількість записів, загальна та середня тривалість сну за цей період. |
| 4️⃣ | **Найкоротша ніч за тиждень**  | Визначається день з найменшою тривалістю сну за останні 7 днів. Виводиться дата, час засинання/пробудження та тривалість. |
| 5️⃣ | **Редагувати запис**          | Користувачка може змінити час засинання та/або пробудження для будь-якого запису. Тривалість перераховується автоматично.  |
| 6️⃣ | **Видалити запис**               | Користувачка може видалити будь-який запис за датою.                                                                    |
-------------------------------------------------------------------------------------------------------------------------------------------------------------------
> 💡 **Додатково:** програма перевіряє коректність введених даних (формат дати, часу) та не дозволяє додавати записи з майбутньою датою.

## 🚀 Як встановити та запустити
### Вимоги до системи
- **Python 3.10** або вище
- Будь-яка операційна система (Windows, macOS, Linux)
### Інструкція з запуску
1. **Склонуйте репозиторій** або завантажте файли проєкту:
   ```bash
   git clone https://github.com/s1060755-star/sleep-diary-tracker.git
   ```
2. **Перейдіть у папку з проєктом:**
   ```bash
   cd sleep-diary-tracker
   ```
3. **Запустіть програму:**
   ```bash
   python main.py
   ```
4. **Користуйтеся меню** — обирайте потрібну дію, вводячи відповідну цифру (1–6) та натискаючи Enter.

## 📁 Формат збереження даних
Усі записи зберігаються у файлі `sleep_data.json` у форматі JSON. Це забезпечує зручність читання, редагування та сумісність з іншими програмами.
**Структура одного запису:**
```json
{
  "date": "2025-06-15",
  "sleep_time": "23:30",
  "wake_time": "07:15",
  "duration_hours": 7.75
}
```
**Приклад повного файлу з кількома записами:**
```json
[
  {
    "date": "2025-06-14",
    "sleep_time": "23:00",
    "wake_time": "07:00",
    "duration_hours": 8.0
  },
  {
    "date": "2025-06-15",
    "sleep_time": "23:30",
    "wake_time": "07:15",
    "duration_hours": 7.75
  }
]
```

## 🛠️ Технічні деталі реалізації
### Структура коду
Програма складається з наступних основних модулів:
| Функція                         | Призначення                                                       |
|---------------------------------|-------------------------------------------------------------------|
| `load_data()`                   | 📥 Завантажує дані з JSON-файлу                                   |
| `save_data()`                   | 💾 Зберігає дані у JSON-файл                                      |
| `validate_time()`               | ⏰ Перевіряє коректність формату часу (HH:MM)                     |
| `validate_date()`               | 📅 Перевіряє коректність формату дати (YYYY-MM-DD)                |
| `calculate_duration()`          | 🧮 Розраховує тривалість сну з урахуванням переходу через опівніч |
| `get_date_from_user()`          | 📅 Запитує дату, поки не буде введено коректну                    |
| `get_time_from_user()`          | ⏰ Запитує час, поки не буде введено коректний                    |
| `get_yes_no()`                  | ❓ Запитує так/ні, повертає True/False                            |
| `add_sleep_record()`            | 🛌 Додає новий запис до щоденника                                 |
| `view_all_records()`            | 📊 Виводить всі записи у вигляді таблиці                          |
| `average_sleep_last_week()`     | 📈 Розраховує середню тривалість сну за останні 7 днів            |
| `shortest_night()`              | 🌙 Знаходить найкоротшу ніч за останні 7 днів                     |
| `edit_record()`                 | ✏️ Редагує існуючий запис                                         |
| `delete_record()`               | 🗑️ Видаляє запис                                                  |
| `show_menu()`                   | 🎨 Виводить головне меню програми                                 |
| `main()`                        | 🚀 Головний цикл програми                                         |
-------------------------------------------------------------------------------------------------------
### Особливості реалізації
- **Обробка помилок:** усі функції введення даних використовують цикли `while True`, які повторюють запит доти, доки користувачка не введе коректні дані. Це запобігає аварійному завершенню програми.
- **Розрахунок тривалості:** якщо час пробудження менший або дорівнює часу засинання, програма автоматично додає 24 години, що дозволяє коректно обробляти випадки, коли людина спить через опівніч.
- **Збереження даних:** усі зміни негайно записуються у файл, тому дані не втрачаються навіть при раптовому закритті програми.
- **Емодзі та візуальне оформлення:** код містить емодзі для кожного розділу, що робить вивід у консолі більш зручним та візуально приємним.
